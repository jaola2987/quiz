JS — Promise .then/.catch
Какое утверждение полностью верно?
// Фрагмент 1
promise.then(f1).catch(f2);
// Фрагмент 2
promise.then(f1, f2);
AОба фрагмента ведут себя одинаково
BВ 1: f2 обрабатывает ошибки только из promise; во 2: из promise и f1
CВ 1: f2 обрабатывает ошибки только из f1; во 2: только из promise
DВ 1: f2 обрабатывает ошибки и из promise, и из f1; во 2: только из promise

let promise = new Promise(function(resolve, reject) {

});
promise.then(console.log).then(console.log);
A1, 2
B2, 1
C1, undefined
D2, undefined
D — верно. Promise может быть resolved только один раз. resolve(2) вызван первым (синхронно) → промис resolved со значением 2. Второй resolve(1) из setTimeout игнорируется. Первый .then получает 2 → выводит 2. Второй .then получает undefined (console.log ничего не возвращает).

JS — Promise / Error в setTimeout
Что произойдёт при выполнении следующего кода?
new Promise(function(resolve, reject) {
  setTimeout(() => {
    throw new Error("Whoops!");
  }, 1000);
}).catch(console.log);
AЧерез 1 секунду промис перейдёт в rejected
BПромис сразу перейдёт в rejected
CЧерез 1 секунду — необработанное исключение (Uncaught Error)
DЧерез 1 секунду в консоль: Error: Whoops!
C — верно. .catch перехватывает ошибки только внутри синхронного executor или явного reject(). throw внутри setTimeout выполняется в другом тике event loop — Promise executor уже завершён. Ошибка «вылетает» за пределы промиса как необработанное исключение.
